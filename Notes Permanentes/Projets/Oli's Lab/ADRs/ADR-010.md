# ADR-010: Google Merchant - mandatory migration from Content API v2.1 to Merchant API v1

## Context

Google is deprecating Content API v2.1 on 18 August 2026. The current implementation uses `googleapis@122.0.0` with `google.content('v2.1')` across ~700 lines in `googleMerchantUtils.js`. Migration is mandatory - there is no choice. The decision is about the target package and implementation strategy.

## Decision

Migrate to `@googleapis/merchantapi@17.3.0`. All 9 current API calls target only the `Products` sub-API, so the migration scope is limited. Do not use `@google-shopping/*` packages (pre-1.0, potentially unstable). Perform Developer Registration (one-time GCP link to Merchant Center account) before any testing.

Key breaking changes to handle:
- Product ID format: `online:en:GB:SKU` becomes `en~GB~SKU` (tildes, no channel prefix)
- Price format: `{ value: "29.90", currency: "EUR" }` becomes `{ amountMicros: 29900000, currencyCode: "EUR" }`
- Update operation: `products.update` becomes `productInputs.patch`
- Insert operation: `products.insert` becomes `productInputs.insert`

## Options Considered

### Option A: @googleapis/merchantapi@17.3.0 (chosen)

- **Pro:** Official stable package (v17.x), same googleapis pattern the team already knows.
- **Con:** Requires handling the breaking changes listed above.

### Option B: @google-shopping/* packages

- **Pro:** More modern API design, modular sub-APIs.
- **Con:** Pre-1.0 versions across all packages. Instability risk for a production-critical integration with a hard deadline.

## Rationale

The deadline is hard. Using the stable official package minimizes risk. The `@google-shopping/*` packages are architecturally cleaner but their pre-1.0 status introduces unpredictable breaking changes at the worst possible time. `@googleapis/merchantapi` is the safe choice.

## Consequences

**Positive**
- Compliance with the 18 August 2026 deadline.
- Merchant API v1 supports page sizes up to 1000 (vs 250 in Content API), which may improve catalogue sync performance.

**Negative / Accepted trade-offs**
- `customBatch` is removed in Merchant API v1 - but the current implementation does not use it, so no impact.
- Product IDs change format in GMC (tilde syntax). Existing GMC product entries may need to be cleared and re-inserted depending on how GMC handles ID format changes.
- Developer Registration must be done once before any API call can succeed - this is a mandatory manual step.
