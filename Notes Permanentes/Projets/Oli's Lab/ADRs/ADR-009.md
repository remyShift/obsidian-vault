# ADR-009: Product feeds - XML on S3 + CloudFront as single marketing integration point

## Context

Three platforms need up-to-date product catalogue data: Google Merchant Center (GMC), Meta Catalog, and Klaviyo. The current situation is broken: the GMC trigger was on Notion (no longer used), Meta is updated via manual Excel export from GMC, and Klaviyo has no catalogue connection at all. The marketing team is blocked. Additionally, Google is sunsetting Content API v2.1 on 18 August 2026 (see ADR-010), which forces a re-architecture regardless.

## Decision

Generate 3 XML feed files (one per platform: GMC, Meta, Klaviyo) from Payload CMS as source of truth, upload them to S3, and distribute via CloudFront with stable URLs. All three platforms read from their respective CloudFront URL. Payload is the trigger: either on product update via a hook, or manually via a CMS button.

## Options Considered

### Option A: Per-platform API push

- **Pro:** Real-time updates.
- **Con:** 3 separate integrations to build and maintain (different auth, different formats, different rate limits). Not scalable as new platforms are added. High maintenance burden.

### Option B: XML feed on S3 + CloudFront (chosen)

- **Pro:** One codebase to maintain per feed format. Meta and Klaviyo both natively support the Google Merchant Center XML format, so one format serves all three. CloudFront provides a stable URL regardless of the underlying S3 file name - no need to update links on Meta, Klaviyo, or GMC when the file is regenerated. Payload is already connected to S3 for media, so the infra is already in place.
- **Con:** Not real-time (minimum refresh: 1h for GMC, 6h for Klaviyo). Stock is limited to in-stock/out-of-stock boolean for now.

## Rationale

The XML feed approach minimizes the number of platform-specific integrations. Meta and Klaviyo both support the Google Merchant Center XML format natively, meaning one well-structured feed file covers both. CloudFront solves the URL stability problem - the file can be regenerated as often as needed without any configuration change on the receiving platforms.

## Consequences

**Positive**
- One feed format to maintain instead of 3 custom integrations.
- Stable CloudFront URLs: Meta, Klaviyo, and GMC configurations never need to be updated.
- Infrastructure already exists (Payload -> S3 connection live for media).
- Supports multi-language from day one (one XML file per language per platform).

**Negative / Accepted trade-offs**
- Not real-time: maximum freshness is 1 hour (GMC) and 6 hours (Klaviyo), which is acceptable for a cosmetics catalogue.
- Stock granularity is limited to boolean (in-stock/out-of-stock). Detailed inventory tracking is out of scope for this phase.
- Feed generation must handle products that are offline/draft to exclude them from the feed.
