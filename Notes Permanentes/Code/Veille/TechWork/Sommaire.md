## Les talks

- [[Le software craftmanship s'est-il perdu (Edouard Cattez)]] : le craft a confondu l'excellence technique avec une fin en soi. Les bonnes pratiques ne valent que par la valeur qu'elles produisent, et il faut les vendre sur ce terrain, pas sur la technique.
- [[Coder sans se noyer dans la complexité (Yoan Thirion)]] : socle scientifique (The Programmer's Brain, Felienne Hermans) sur les limites de nos mémoires. On passe 60% du temps à lire du code, donc écrire du code "chunkable" et bien nommé est rentable. Le plus dense et le plus actionnable.
- [[Comment bien coder avec l'IA (Musso, Teissedre)]] : bien coder avec l'IA n'est pas une question d'outil mais de discipline. Trois piliers, spécification, architecture, feedback loop, qui existaient avant l'IA et que celle-ci rend non négociables.
- [[Pimp my shell (Julien Wittouck)]] : note de setup. Remplacer les binaires par défaut (bat, lsd, zoxide, fd), prompt starship, runtimes via mise, dotfiles via chezmoi.
