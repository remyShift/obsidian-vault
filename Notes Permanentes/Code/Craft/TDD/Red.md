#### Red :

Pour commencer une fonctionnalité on écrit un test qui correspond au comportement attendu mais qui échoue car pas de code encore écrit qui le valide, donc en rouge dans notre IDE.
- Permets de cadrer l’écriture du code de production et ainsi ne pas partir dans tout les sens et avoir une ligne directrice 
	- Importance de bien checker que le test échoue pour les bonnes raisons (vérifier données d’entrées et résultats attendu)

Et si le test écrit est directement vert ?
- C’est généralement mauvais signes et peut être lié à :
	- un tests mal écrit, valeur d’entrées ou de résultats sont fausses,
	- le comportement attendu a déjà été codé lors d’une précédente itération.
