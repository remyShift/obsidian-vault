Dans l’imaginaire collectif **code legacy** = code vieux, mal écrit et pleins de bugs.

**Code Legacy =** Code sans tests
- On s’affranchit de la limite de date pour dire qu’un code est legacy,
- Sans tests difficile de savoir le comportement attendu de notre code,
- Modifier du code sans tests ne nous permets pas de savoir si on touche à d’autres choses sensibles // si on va ajouter des régressions,
- Point de départ lorsqu’on travail sur du code legacy : ajouter des tests,

Voyons maintenant comment [[Travailler sur du Code Legacy]].

En conclusion il faut veiller à ne pas paniquer, limiter les risques et se placer en position de confiance par rapport au code.
