Si le code est relativement claire et peu volumineux on peut commencer à introduire des tests basés sur notre compréhension.
- Mais par où commencer ?

« Tester en premier les branches les moins profondes, refactorer en premier les branches les plus profondes. », *SoftwareCraft mais citation de Sandro Mancuso*

Ce qui veut dire qu’on veut tester dans un premier temps ce qui nous est le plus accessible —> on aura ainsi plus de facilité à écrire les tests.
- En suivant cette quote on va plus refactorer la profondeur d’une fonction plutôt que la fonction dans sa globalité ce qui élimine un peu de sa complexité globale. (*cf exemple du livre p96*)
