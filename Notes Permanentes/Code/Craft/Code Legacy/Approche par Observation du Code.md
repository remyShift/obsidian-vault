**NB :** pour cette note se référer à la résolution du Gilded Rose Kata dans le livre SoftwareCraft p97.

La plupart du temps le code est tellement complexe que c’est cette méthode qui sera utilisée.

On peu à la première lecture noter tout ce qui nous saute aux yeux et ne rentre pas dans le clean code :
- corps de fonctions trop long,
- mauvais nommage,
- trop de paramètres d’entré,
- duplication,
- trop de succession de if, 
- etc …

Néanmoins on ne perdra pas trop de temps la dessus et on préférera se construire un [[Golden Master]].
