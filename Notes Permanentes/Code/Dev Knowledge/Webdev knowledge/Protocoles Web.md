- Différents protocoles régissent les connections entre appareils ou à certains réseaux comme Internet, ils sont évidemment appliqué aux [[Pages Web]] pour gérer leur relation [[Organisation Client Serveur |client / serveur]].

##### HTTP (**H**yper**T**ext **T**ransfer **P**rotocol) :
- Protocole de communication pour transférer des données sur le **W**orld **W**ide **W**eb (www)
	- Facilite la demande et transmission de pages web entre client et serveur.
##### TCP (**T**ransmission **C**ontrol **P**rotocol) :
- Protocole de transport qui assure un réseau fiable et ordonné entre 2 appareils sur un même réseau. Lorsque tu envoies des données sur Internet, TCP divise ces données en petits paquets et s'assure qu'ils arrivent à destination dans le bon ordre et sans erreur. Si l'un des paquets est perdu en cours de route, TCP le détecte et le renvoie pour s'assurer que toutes les données sont bien reçues.
##### IP (**I**nternet **P**rotocol) :
- L'adresse IP = adresse postale et le protocol IP = le facteur,
	- Protocole de réseau responsable de l'acheminement des paquets de données entre les appareils en les distribuant aux bons appareils selon leur [[Adresses de connection#Adresse ip |adresse ip]].
##### API (**A**pplication **P**rogramming **I**nterface) :
- Ensemble de règles et protocoles permettant à des logiciels de communiquer entre eux.
##### URL (**U**niform **R**esource **L**ocator) :
- Adresse utilisée pour identifier de manière explicite une ressource prise en charge par le protocole HTTP // adresse pour localiser une [[Pages Web]] sur [[Internet]].
