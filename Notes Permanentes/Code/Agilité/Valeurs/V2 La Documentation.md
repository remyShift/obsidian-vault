### 2. Des logiciels opérationnels plus qu’une documentation exhaustive

- La documentation c’est la vie néanmoins de la bonne doc
- Mais une équipe de dev devrait plutôt se concentrer sur la production de fonctionnalités plutôt que de la rédaction de documents

- Les méthodes de gestion de projet classique peuvent être parfois laborieuse quant à la gestion de la doc associé, les chefs de projets peuvent passer des heures à mettre à jour des présentations ou des calendriers.
- La méthode Agile va permettre d’avoir des modèles de documents qui sont toujours les mêmes et qui se complètent rapidement avec l’essentiel des infos.

- 2 type de docs agile :
	- La doc sur les exigences du produit : la vision du produit, la roadmap produit, les besoins utilisateurs et spécifications fonctionnelles,
	- La doc sur les spécifications techniques : l’architecture du logicielle / matérielle, les procédures d’installations etc
