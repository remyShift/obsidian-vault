Si tu codes juste pour que “ça marche”, tu passes à côté de l’essentiel. 👇🏼


Faire marcher un programme, c’est la base.
Mais un bon développeur va plus loin :

- Il code pour les utilisateurs, pas juste pour le compilateur,

- Il code pour que ça dure, pas juste pour la démo de demain,

- Il code pour que ce soit lisible, même dans 6 mois à 2h du matin,

- Il code pour ses collègues, ceux qui maintiendront son code après lui.

Coder pour que ça marche, c’est un point de départ.
Pas un objectif.

« Working code is low bar. »

👉🏼Et toi tu codes pour que ça marche ou tu vas plus loin ?
