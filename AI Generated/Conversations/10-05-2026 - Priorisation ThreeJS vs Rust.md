# 10-05-2026 - Priorisation ThreeJS vs Rust

**Domaine :** Code / Créatif
**Statut :** Synthèse brute -> à intégrer dans les notes permanentes si pertinent

---

## Contexte
Rémy a commencé deux technos en parallèle : Rust et ThreeJS. Il ne voulait pas se disperser et cherchait à savoir laquelle prioriser, en tenant compte du marché de l'emploi actuel et de son contexte personnel (départ à Séoul dans quelques mois, profil fullstack JS junior, objectif Awwwards).

---

## Points clés

- **Objectif principal 1-2 ans :** maximiser les options pour partir à Séoul
- **Raison ThreeJS :** curiosité pure + objectif concret d'avoir un site sur Awwwards / créer une vraie expérience web époustouflante
- **Raison Rust :** tendance du marché, mais aussi logique de progression - vouloir s'élever via un langage fortement typé pour développer une rigueur de dev que JS/TS ne force pas naturellement
- **Verdict :** ThreeJS maintenant, Rust plus tard (pas de Go entre les deux)

**Sur le marché :**
- Rust : salaires élevés (130k$ moyenne, 235k$ senior), +35% d'offres en 1 an, mais demande une spécialisation éloignée du profil actuel (systems, blockchain, infra cloud)
- ThreeJS : niche mais cohérente avec le profil JS/React existant, portable à l'international, forte scène créative en Corée du Sud
- Les deux sont en croissance, mais ThreeJS s'intègre directement dans la trajectoire actuelle

**Sur l'argument rigueur / langage fortement typé :**
- L'argument est légitime et reconnu - pas juste du suivisme
- Go aurait été plus adapté à court terme (courbe moins brutale, concepts transférables)
- Rust reste le choix de Rémy malgré tout - décision assumée
- Point important : Rust ne peut pas être un apprentissage "en parallèle", ça devra être un focus dédié le moment venu

**Stack creative dev à construire autour de ThreeJS :**
- ThreeJS + GLSL (shaders) + GSAP + sensibilité design
- Référence : Bruno Simon (français, Paris, créateur de Three.js Journey, multiple SOTD/SOTM Awwwards)

---

## Décisions prises

- Prioriser ThreeJS maintenant
- Mettre Rust en pause - reprendre plus tard comme focus principal dédié
- Pas de Go en intermédiaire
- Commencer / continuer Three.js Journey de Bruno Simon si pas encore fait
