## Problèmes / Bugs
- 🔴 2FA totalement bypassable — `apps/web/src/app/api/settings/security/2fa/verify/route.ts:58` : `const isValid = /^\d{6}$/.test(code)` accepte n'importe quel code à 6 chiffres alors qu'`otplib` est installé
- 🔴 `middleware-security.ts` exporte `securityMiddleware` jamais appelé depuis `middleware.ts` → CSP, HSTS, X-Frame-Options absents en prod
- 🔴 Routes `/api/test` (sans auth) et `/api/test-stats` exposées en prod
- 🔴 Rate limiter en mémoire (`Map<string, RateLimitEntry>`) alors que `@upstash/ratelimit` est installé — bypass trivial sur Vercel multi-instance
- 🔴 12 versions du wallet schema SQL coexistent (`wallet_clean`, `wallet_complete`, `wallet_final`, `wallet_fix`, etc.) — source de vérité inconnue
- 🔴 `next.config.js` : `images.remotePatterns: [{ hostname: '**' }]` accepte tout HTTPS (SSRF, abus optimization)
- 🔴 Webhook Viva : HMAC vérifié mais pas d'idempotence anti-replay → double crédit possible
- 🟠 `wallet/withdraw/route.ts` : `validateWithdrawal(supabase: any, ...)` + TOCTOU sur balance (race condition)
- 🟠 Drift de versions web↔admin : Zod 3 vs Zod 4 (incompatible), Tailwind 3 vs 4, `bcrypt` natif vs `bcryptjs`
- 🟠 ESLint cassé : `eslint@10.0.1` (n'existe pas) dans web, `eslint@4.0.0` (2017) dans admin, `eslint-config-next@0.2.4` (mauvais package)
- 🟠 `package-lock.json` ET `pnpm-lock.yaml` cohabitent, `pnpm-lock.yaml` untracked
- 🟠 `packages/shared-types/` vide ET hors workspace (`pnpm-workspace.yaml` ne liste que `apps/*`)
- 🟠 1 seul commit "Initial commit" pour tout le projet, 0 branche, `.windsurf/rules` supprimé non committé
- 🟠 0 test unitaire, 0 test intégration, 1 placeholder Playwright pour 142 routes API
- 🟠 Aucun `.github/`, aucun hook pre-commit, aucun Dependabot
- 🟠 Aucun monitoring (pas de Sentry) + `removeConsole: production` strippe tous les `console.error` → aucun log d'erreur en prod
- 🟠 Composants monstres : TeamsClient.tsx 1049L, ProfileOverview.tsx 880L, ranked/[id]/page.tsx 868L, winamatch/[id]/page.tsx 856L
- 🟠 38 `: any` + ~50 `as any` dans apps/web/src malgré `strict: true`
- 🟠 RGPD partiel : routes delete-account/data-export OK, mais pas de cookie banner, pas de mentions légales, pas d'age gate 18+
- 🔴 Statut juridique gambling/skill gaming non tranché : plateforme avec wallet/withdraw/winamatch en argent réel sans licence ANJ ni vérification d'âge

## Découvertes
- Le projet a été poussé en 1 seul commit "Initial commit — Winalia platform" sans historique (potentiellement migré depuis un autre repo ou copié-collé en bloc)
- La logique métier est massivement en PL/pgSQL (RPC `complete_match`, `record_ranked_match_result`, `add_to_wallet`, `subtract_from_wallet`) → modèle anémique côté TS, logique non testable en TS
- `@supabase/auth-helpers-nextjs` au root + dans apps/web alors que `@supabase/ssr` (le remplaçant officiel) est aussi présent → cohabitation deprecated/moderne
- `apps/web/package.json` force `next dev --webpack` au lieu de Turbopack par défaut (Next 16) → dev plus lent
- Présence de scripts ad-hoc à la racine de apps/web : `fix-all-params.js`, `fix-params-await.sh`, `fix-routes.sh`, `check-env.js` + 4 HTML de debug (`debug-test.html`, `test-local.html`, `test-simple.html`, `test-supabase.html`)
- Le README mentionne `PRODUCTION_CHECKLIST.md`, `SUPABASE_HEADER_REQUIREMENTS.md`, dossier `supabase/` qui n'existent pas
- 22 bounded contexts naturels détectés (wallet, auth, ranked, winamatch, shop, teams, profile, watch, premium, battle-pass, affiliates, referrals, inventory, notifications, settings, dashboard, rankings, stats, users, user, matches) sans aucune isolation
- Bon point : tsconfig.base strict, validation Zod ~85%, hCaptcha présent, KYC workflow présent, Radix UI partout (a11y gratuite), webhooks signés (Viva HMAC SHA256, Xsolla auth header)
- Bon point : routes RGPD `delete-account` et `data-export` déjà codées
