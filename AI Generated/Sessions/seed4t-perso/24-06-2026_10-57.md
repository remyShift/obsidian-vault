## Découvertes
- **Builder** : `add` accumule, `build` valide. La dédup catalogue a sa place dans `build`, pas dans `add`.
- **Le comportement va où vit la donnée** : la résolution appartient au `Catalog` (il possède les arêtes), pas au `Cart`.
- **Récursion = 3 piliers** : geste répété + frein (`visited`) + condition d'arrêt.
- **`arrayContaining` teste la présence, jamais l'absence ni les doublons** → faux ami récurrent.
- **Un test jamais vu rouge n'est pas fiable** → systématiser le rouge avant le vert.
- **Modèle/vue** : stocker l'entrée irréductible (`roots`), dériver la sortie (`bricks`).
- **Pre-order = ordre de sélection** ; post-order = ordre d'install/topologique.
- **Dépendances par nom (`string[]`) = comme un package.json** : une référence (clé étrangère vers le catalog), pas l'objet copié. Copier l'objet = dupliquer la vérité = le bug.
- **Distinction produit clé** : les transitives npm classiques vont dans `node_modules`/lockfile, jamais dans le `package.json`. Ce qui apparaît vraiment et "tire plusieurs" = peer deps + grappes d'outillage, et c'est **plat** (1 niveau). La transitivité profonde + diamants + cycles du kata = bon terrain TDD mais hors besoin produit.
- **Monorepo pnpm** : un rename de dossier de package désynchronise le lockfile (indexé par chemin) → CI casse à `--frozen-lockfile`. Réflexe : `pnpm install` après tout rename/déplacement.
- **eslint interdit `require`** même en `.js` (`no-require-imports`) → script CI écrit en `.mjs` (ESM) + `readFileSync` au lieu de `require`.
- Mismatch à corriger : dossier `packages/domain` mais nom de package toujours `@seed4t/core`.
