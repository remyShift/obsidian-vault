## Problèmes / Bugs
- Citation systématiquement tronquée à « invincible » : faux diagnostic initial (largeur du bloc, testé 88 -> 100 -> 64 -> fill sans effet). Vraie cause : le renderer `animated_typewriter` tape dans la fenêtre d'animation de 2s et fige avant la fin sur les citations longues (88 caractères = trop long pour finir en 2s).
- 2s d'attente avant de récupérer le terminal, non résolu par `duration_ms = 500` : la fenêtre 2s est indépendante de la vitesse d'animation.
- Splash global parasite dans les sous-dossiers d'un repo : le hook cd lançait `splashboard` nu, qui ne résout le dashboard project qu'à la racine du repo.

## Découvertes
- `ANIMATION_WINDOW` de 2s **codée en dur** dans splashboard 2.9.0 (vu dans les strings du binaire) : dès qu'un seul widget est animé, le terminal est gelé 2s. `duration_ms` ne change que la vitesse de l'animation, pas le blocage. Aucun réglage global (seules clés `[general]` : auto_home, auto_on_cd, wait_for_fresh, flush_to_scrollback, legacy, subtle, height, locale).
- `random_quote` renvoie la citation en une seule chaîne et splashboard n'a aucun word-wrap (grep `wrap` sur tout le catalog = vide) : impossible de mettre la citation sur 2 lignes nativement.
- `text_ascii` = renderer figlet **statique** (options `style = figlet`, `font`, `color`, `align`) ; `text_plain` = texte statique aligné. Le morph animé `animated_figlet_morph` forwarde justement vers `text_ascii`.
- Résolution dashboard splashboard = **racine de repo uniquement** (sinon fallback home). Override per-dir possible via un `.splashboard.toml` / `.splashboard/dashboard.toml` au root (résolu en remontant l'arbo).
- `random_fortune` peut être multi-lignes (haikus, verses) mais ce sont des fortune cookies anonymes, pas des citations attribuées.
- Confirmé : le Bash de Claude (zsh figé sans TTY) ne peut pas rendre les TUI -> itération par screenshots de Rémy à chaque changement visuel.

## Décisions prises
- **Tout statique, par choix** : priorité au terminal instantané plutôt qu'à l'animation. La fenêtre 2s non contournable a tranché le compromis (validé par Rémy via question).
- Citation laissée en **mono-ligne** `text_plain` centrée pleine largeur. Refus de passer à `random_fortune` pour gagner le multi-lignes : ça dénaturerait le widget.
- Hook cd en `--on-cd` : pas de splash home parasite dans les sous-dossiers ; le splash home n'apparaît plus qu'au lancement d'un shell.
