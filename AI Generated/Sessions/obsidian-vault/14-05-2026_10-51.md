# Recap — obsidian-vault — 14/05/2026

## Avancées
- Correction de la query Dataview "Notes totalement déconnectées" dans Sommaire.md : les notes avec uniquement des outlinks vers des médias (images, etc.) sont maintenant correctement détectées comme orphelines
- Solution finale validée et fonctionnelle : `all(file.outlinks, (l) => contains(string(l), ".png") OR ...)`

## Problèmes / Bugs
- Bug Dataview : `length(file.outlinks) = 0` excluait les notes contenant uniquement des images embarquées (`![[image.png]]`), car Dataview les comptait comme des outlinks
- `l.path` retourne `null` pour les outlinks pointant vers des fichiers non-markdown (images) — toutes les approches basées sur `l.path` ont échoué silencieusement
- `regexmatch` avec `l.path` null retourne `false` sans erreur, ce qui rendait le filtre inopérant
- `filter() + length()` ne fonctionnait pas correctement avec `l.path` null dans les lambdas Dataview

## Découvertes
- Dans Dataview, `l.path` est `null` pour les liens vers des fichiers non-markdown (images, PDFs, etc.). Il ne faut jamais utiliser `l.path` pour filtrer les outlinks qui peuvent inclure des médias.
- `string(l)` est la seule méthode fiable pour obtenir le nom/chemin d'un outlink quel que soit son type
- `all(array, predicate)` existe bien en Dataview DQL et fonctionne correctement
- `any(array, predicate)` est l'alternative à `all()` — `!any(...)` est équivalent à `all(!...)
- La query de debug `TABLE map(file.outlinks, (l) => string(l))` est utile pour inspecter les outlinks réels

## Décisions prises
- Query "Notes totalement déconnectées" : utilise `string(l)` au lieu de `l.path` dans les lambdas Dataview
- Pattern retenu : `length(file.inlinks) = 0 AND (length(file.outlinks) = 0 OR all(file.outlinks, (l) => contains(string(l), ".ext") OR ...))`
- Extensions filtrées : png, jpg, jpeg, gif, svg, webp, bmp, tiff, mp4, mp3, pdf

## Prochaines étapes
- Aucune action vault spécifique prioritaire
- Si besoin de filtrer des outlinks par type dans d'autres queries Dataview, toujours utiliser `string(l)` et non `l.path`
