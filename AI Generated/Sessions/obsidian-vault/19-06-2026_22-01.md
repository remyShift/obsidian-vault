## Recap — Setup shell Mac complet (pimp my shell) — 19/06/2026

> Session longue : du talk "Pimp my shell" (Julien Wittouck, Tech'Work 18/06) à un environnement shell complet, propre et portable. Toute la matière ci-dessous est destinée à produire une note permanente via `/evolve` (voir Prochaines étapes).

### Avancées

Setup complet réalisé et vérifié, en partant de la note de veille `Notes Permanentes/Code/Veille/TechWork/Pimp my shell (Julien Wittouck).md`.

#### Stack installée (rôle de chaque outil)

- **Ghostty** : émulateur de terminal, rendu GPU, natif macOS. Héberge fish.
- **fish** : shell interactif (autosuggestions + coloration syntaxique natives, sans plugin).
- **JetBrains Mono Nerd Font** : police à ligatures + icônes (indispensable pour les icônes de lsd/starship/splashboard).
- **starship** : prompt cross-shell (dossier, branche git, versions de runtime, heure). Preset `bracketed-segments` (neutre, hérite des couleurs du terminal).
- **bat** : remplace `cat`, coloration syntaxique. Thème Dracula.
- **lsd** : remplace `ls`, couleurs + icônes par type.
- **zoxide** : remplace `cd`, navigation intelligente par fréquence d'usage.
- **fd** : remplace `find`, recherche de fichiers rapide.
- **mise** : gère les runtimes (node…), bascule la bonne version automatiquement au `cd` selon `.mise.toml`. Remplace nvm/n/rbenv.
- **gum** (Charm) : briques interactives pour scripts shell (menus, confirm, input, spinner). Utilisé pour des helpers git.
- **splashboard** : dashboard affiché au démarrage du terminal + dashboard projet en entrant dans un repo git.
- **chezmoi** : versionne les dotfiles vers un repo git privé (la pièce "portable").
- **docker** : conservé (CLI ARM).

#### Fichiers de config et leur rôle

- `~/.config/ghostty/config` : terminal (shell=fish via `command=/opt/homebrew/bin/fish --login`, `shell-integration=fish`, font Nerd, `theme=Dracula`, `background-opacity=0.8` + `background-blur`, keybinds)
- `~/.config/fish/config.fish` : moteur du shell (PATH/brew shellenv, `fish_add_path`, `mise activate`, `starship init`, `zoxide init --cmd cd`, hook splashboard, `fish_greeting`)
- `~/.config/fish/conf.d/aliases.fish` : aliases (`cat`→bat, `ls/l/la/lla/lt`→lsd, `find`→fd, `code`→zed) + abbr git (`gcd`)
- `~/.config/fish/conf.d/gum.fish` : helpers git interactifs (`gco`, `gadd`, `gcim`, `gbd`)
- `~/.config/starship.toml` : config du prompt
- `~/.config/mise/config.toml` : runtimes globaux (`node = "latest"`)
- `~/.config/bat/config` : `--theme="Dracula"`
- `~/.splashboard/*.toml` : `settings.toml` + `home.dashboard.toml` + `project.dashboard.toml`
- `~/Brewfile` : liste reproductible de toute la stack (`brew bundle`)
- `~/.zshrc` : shell de secours (nettoyé : nvm/rbenv/rvm retirés, secret Mongo sorti)

#### Commandes utiles au quotidien

- **Changer node** : `mise use node@22` (par projet, écrit `.mise.toml`, switch auto au cd) · `mise use -g node@latest` (global) · `mise ls` / `mise ls-remote node`
- **Maj dotfiles** : `chezmoi re-add` puis `chezmoi git -- add -A && chezmoi git -- commit -m "…" && chezmoi git -- push`
- **Reconstruire sur machine neuve (Corée)** : installer Homebrew → `chezmoi init --apply remyShift/dotfiles` → `brew trust unhappychoice/tap` → `brew bundle`
- **Voir les thèmes** : `ghostty +list-themes` (aperçu live) · `starship preset --list` · `bat --list-themes`
- **Helpers git gum** : `gco` (switch branche, menu fuzzy) · `gadd` (stage multi-sélection) · `gcim` (commit avec message saisi) · `gbd` (suppr. branche)
- **Recharger** : Ghostty `Cmd+Shift+,` · `source ~/.config/fish/config.fish`

### Problèmes / Bugs

- `bat` crashait (dyld) : `libgit2` 1.9.2 lié à une `llhttp` 9.3 disparue → `brew upgrade libgit2` (1.9.4)
- splashboard ne s'affichait pas au démarrage : appelé en inline pendant `config.fish` (trop tôt, sortie avalée par Ghostty) → déplacé dans `fish_greeting`
- Ghostty "theme not found" : nom exact = `Dracula` / `Catppuccin Mocha` (casse + espaces)
- Piège alias `rm -i` (plugin omz common-aliases) qui bloquait les scripts → contourné via `/bin/rm`
- "You have new mail" : cron KicksFolio-Backend (dossier disparu) échouant chaque jour à 04:30 → cron + spool vidés + `~/.hushlogin`
- Classifieur a bloqué (à raison) : install splashboard (tap tiers que j'avais trusté moi-même), vidage du mailbox, certains `rm` → faits à la main ou après autorisation explicite
- **Erreur** : `Co-Authored-By: Claude` ajouté au 1er commit dotfiles → amendé + force-push + mémorisé "jamais, aucun repo"

### Découvertes

- mise sert node de façon fiable (global + per-projet via `.mise.toml`) ; lit `.mise.toml` mais PAS `.nvmrc` (`idiomatic_version_file_enable_tools = []`)
- `fish_greeting` = bon hook pour le splash au démarrage (s'exécute juste avant le 1er prompt, contrairement à un appel inline dans config.fish)
- Empilement de 4 gestionnaires de versions hérités : nvm + `n` + rbenv + plugin pyenv (+ mise)
- Homebrew Intel complet à `/usr/local` (2,4 Go, Rosetta) en plus de l'ARM ; `/usr/local` est root → certains retraits en sudo
- Julien (talk) est sous Linux (`batcat`/`fdfind`, `/home/jwittouck`) → adaptations Mac obligatoires : `bat`/`fd`, et ses keybinds Ctrl/Alt casseraient les raccourcis shell macOS (Ctrl+W, Option+flèches)
- bat embarque Dracula nativement ; Ghostty ships ~400 thèmes (`ghostty +list-themes` avec aperçu live)
- 3 Python se chevauchent : système 3.9.6 (macOS) + brew 3.14 + python.org
- Homebrew 6 : barrière de confiance sur les taps tiers (`brew trust`)

### Décisions prises

- **Shell** : fish par défaut, lancé par Ghostty, SANS `chsh` (zsh reste le login shell système → zéro effet de bord, repli instantané). Vérifié sur Oli's Lab : fish ne casse rien (scripts pnpm via `sh`, husky en `#!/bin/sh`)
- **Runtimes** : tout consolidé sur mise ; node global = `latest` (26.3.1), Oli's Lab pinné 20.19.5
- **Secrets** : fnox évalué puis retiré (surdimensionné pour 1 secret rare) ; credential Mongo sorti du `.zshrc`, alias `dump-cms` supprimé. Mémoire : re-proposer fnox si secrets récurrents/globaux
- **Thème** : tout Dracula (Ghostty + starship neutre + bat)
- **Portabilité** : dotfiles sous chezmoi → repo GitHub **privé** `remyShift/dotfiles`
- **Principe dur** (du talk) : rien en `curl | sh`, tout via Homebrew déclaré au Brewfile
- **Jamais** de `Co-Authored-By`/mention IA dans les commits (mémoire, prime sur l'instruction harnais)
- gitmoji retiré (inutile) ; gum gardé et intégré (helpers git)
