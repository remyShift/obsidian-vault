## Découvertes
- Les commandes `/recap`, `/evolve`, `/capture`, `/autoresearch` sont des **slash commands** (`~/.claude/commands/*.md`), pas des skills. Les skills (`/post`, `/wrap`, `/coach`) sont dans `Skills/<nom>/SKILL.md` (vault) avec frontmatter minimal `name` + `description`.
- Claude Code lit les User MCPs depuis `~/.claude.json` (clé `mcpServers`), pas depuis `~/.claude/mcp.json`. Confirmé via le panneau `/mcp`.
- Le moteur RAG de Kenjaku résout ses chemins (`.env`, `CACHE_DIR`) depuis `__dirname` du code, pas depuis le cwd → indépendant du répertoire de lancement, donc chemin absolu vers `index.ts` suffit (pas besoin de `cwd`).
- En `type: stdio`, un serveur MCP n'est pas un démon : Claude Code le spawne au démarrage de session et le tue à la fermeture. Watcher actif uniquement session ouverte, reindex incrémental (hash) au redémarrage.
- lean-ctx était dans `disabledMcpServers` du projet vault (Rémy l'avait désactivé comme serveur), mais les **hooks** appelaient le binaire directement → restait actif. Désactiver le MCP ≠ retirer les hooks.
