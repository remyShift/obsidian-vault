# Recap — obsidian-vault — 19/06/2026

## Avancées
- Diagnostiqué l'origine du message "mgrep CLI n'est pas installé" vu dans une autre session : plugin `mgrep@Mixedbread-Grep` (scope user) installé mais binaire `mgrep` absent du PATH.
- Désinstallé le plugin via la CLI non-interactive `claude plugin uninstall mgrep@Mixedbread-Grep -y` (plus sûr que d'éditer `~/.claude.json` à la main).
- Retiré la marketplace source `Mixedbread-Grep` (`claude plugin marketplace remove`) pour éviter une réinstallation accidentelle.
- Nettoyé les résidus `/tmp` (`mgrep-watch-pid-*.txt`, `mgrep-watch-command-*.log`).

## Problèmes / Bugs
- Hook SessionStart du plugin ([mgrep_watch.py](AI Generated/Sessions/obsidian-vault/19-06-2026_18-17.md)) lançait `subprocess.Popen(["mgrep", "watch"])` à chaque session → échec silencieux (binaire absent), écriture d'un PID file + log vide dans `/tmp`, ET injection quand même de "CRITICAL: you MUST use mgrep" dans le contexte → ordre d'utiliser un outil inexistant.
- Le skill mgrep est un override agressif ("MANDATORY, replaces ALL search tools, NEVER use Grep/Glob/WebSearch") : hijacke toute recherche dans tous les projets, même quand le grep littéral est le bon outil.

## Découvertes
- `claude plugin` expose une CLI non-interactive complète (`list`, `disable`, `uninstall`, `marketplace remove`, etc.) → pas besoin du menu `/plugin` ni d'éditer `~/.claude.json`.
- `mgrep` (Mixedbread) = "semantic grep" SaaS-backed : indexe/embed les fichiers, cherche par le sens ; `--web --answer` tape leur store hébergé. Donc envoi de requêtes/contenu chez un tiers — red flag sur code client sous NDA.
- Les changements de plugin demandent un restart pour s'appliquer pleinement (l'injection peut traîner dans la session courante déjà ouverte).

## Décisions prises
- Ne PAS réinstaller mgrep : redondant avec `vault-rag` (sémantique local, gratuit, déjà opérationnel) pour le vault ; Grep + agent Explore suffisent pour le code ; risque confidentialité + skill toxique.
- Suppression complète : plugin + marketplace + résidus `/tmp`.
