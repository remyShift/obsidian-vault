## Découvertes
- **Orpheline Dataview ≠ orpheline graphe Obsidian** : `file.inlinks = 0` compte les backlinks *entrants* uniquement, alors que la vue graphe affiche les edges dans les deux sens. Une note avec outlinks mais zéro inlinks apparaît "connectée" dans la graphe tout en étant orpheline au sens Zettelkasten. C'est exactement ce qu'on veut chasser : les notes que personne ne cite, donc non redécouvrables par backlinks.
- **Remote Claude Code triggers = cloud sandbox** : les scheduled agents (`RemoteTrigger`) tournent dans un git clone isolé, sans accès à `~/.claude/`. Donc un `/evolve --dry-run` distant ne peut PAS comparer les patterns détectés à l'état actuel des memory files — l'analyse perdrait 50% de sa valeur (propose des trucs déjà en place, rate les contradictions)
- **Auto-commit Obsidian ≠ auto-push** : le plugin commit toutes les 60s localement mais ne pushe probablement pas automatiquement → un remote trigger lirait des recaps potentiellement stale
- `/recap` crée les recaps dans `Obsidian Vault/AI Generated/Sessions/{project}/` et merge par date — pas besoin de hook Stop pour la persistance si on est discipliné

## Décisions prises
- **Retrait du hook `Stop`** : le recap bash-only était un log, pas une synthèse. Discipline `/recap` manuel > filet de sécurité pauvre qui pollue `Sessions/` avec des doublons
- **Suppression de `/link` et `/normalize-frontmatter`** : une commande par besoin réel, pas par possibilité théorique. Moins de surface à maintenir
- **`/lint --fix` volontairement étroit** : automatise uniquement les intégrations AI→Permanentes + régénération sommaires. Toutes les autres catégories (suppression de notes vides, archivage contenu obsolète, promotion Inbox) restent manuelles car trop ambiguës
- **Pas de cron pour `/evolve`** : discipline manuelle le vendredi. Rejet des 3 alternatives :
  - Remote trigger → accès data compromis (pas de `~/.claude/` visible)
  - Launchd local → consomme quand même le Pro plan
  - Committer memory files dans le vault repo → duplication + exposition
- **Design `/evolve` en 2 phases** : `--dry-run` sépare l'analyse coûteuse (vendredi soir, fatigué) de l'application (samedi matin, frais, 3 min) → gain de qualité même sans automatisation
