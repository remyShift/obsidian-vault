# Recap — obsidian-vault — 12/04/2026

## Avancées
- Connexion MCP entre Claude Desktop et le vault Obsidian fonctionnelle via `@modelcontextprotocol/server-filesystem`
- Commande `/recap` créée (`~/.claude/commands/recap.md`) pour générer des résumés de session structurés et les sauvegarder dans Obsidian
- Hook `SessionStart` configuré dans `~/.claude/settings.json` — charge automatiquement les 3 derniers recaps du projet courant au démarrage de Claude Code
- Script `~/.claude/scripts/session-context.sh` créé pour injecter le contexte des sessions précédentes
- Structure `Sessions/{nom-projet}/` mise en place dans le vault Obsidian (sous-dossiers par projet)

## Problèmes / Bugs
- Le package `mcp-obsidian` (via npx) avait des `inputSchema` vides — Claude Desktop voyait le connecteur mais ne pouvait pas utiliser les outils (`read_notes`, `search_notes` sans paramètres définis)
- La config initiale utilisait `npx` comme commande, mais Claude Desktop ne chargeait pas le PATH de nvm/pnpm — le serveur ne démarrait pas (aucune icône MCP visible)
- Après passage au chemin absolu de node, le serveur démarrait (logs OK) mais les schemas restaient cassés côté `mcp-obsidian`

## Découvertes
- Claude Desktop résout son propre PATH pour les MCP servers (logs montrent qu'il trouve nvm) — le problème n'était pas le PATH mais les schemas du package `mcp-obsidian`
- Les logs MCP sont dans `~/Library/Logs/Claude/mcp-server-obsidian.log` — très utiles pour le debug
- Le plugin Obsidian `mcp-tools` (v0.2.27) était aussi installé dans le vault mais n'était pas utilisé par Claude Desktop
- Les hooks Claude Code reçoivent un JSON sur stdin avec `session_id`, `transcript_path`, `cwd` — accès complet au transcript via fichier JSONL

## Décisions prises
- Remplacement de `mcp-obsidian` par `@modelcontextprotocol/server-filesystem` (serveur officiel Anthropic, schemas propres)
- Approche commande `/recap` plutôt que hook automatique `SessionEnd` — Claude génère un meilleur résumé avec le contexte complet de la conversation
- Organisation en sous-dossiers par projet dans `Sessions/` plutôt qu'un dossier plat avec noms de fichiers préfixés

## Prochaines étapes
- Tester le hook `SessionStart` en ouvrant Claude Code sur un projet qui a des recaps
- Valider que le recap apparaît correctement dans Obsidian (frontmatter, tags, rendu)
- Éventuellement ajouter un hook `SessionEnd` léger comme filet de sécurité (rappel de faire `/recap`)
