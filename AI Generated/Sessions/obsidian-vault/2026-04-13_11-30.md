## Problèmes / Bugs
- `git add` a échoué après `git init` à cause d'un fichier `index.lock` — résolu en supprimant le lock
- `rm` sans `-f` demandait une confirmation interactive en mode shell non-interactif — résolu avec `rm -rf`
- Warning GitHub : le fichier `mcp-tools/bin/mcp-server` fait 58 MB (au-dessus du seuil recommandé de 50 MB) — non bloquant mais à surveiller

## Découvertes
- Le setup existant (hook SessionStart + /recap + auto-memory) est plus avancé que ce que proposent les vidéos — aucune n'a de mécanisme de continuité de session automatique
- L'approche Karpathy (wiki LLM avec index centralisé) a un problème de scaling linéaire des tokens — bon à savoir mais pas critique avec un vault petit
- Pinecone est pertinent uniquement pour les gros corpus statiques (transcripts, archives) — pas prioritaire actuellement
- Le modèle mental à 3 couches (CLAUDE.md = identité, Obsidian = raisonnement, Pinecone = recall) est solide
- Le web clipper Obsidian est une extension navigateur officielle, pas un plugin vault

## Décisions prises
- Tout le contenu AI-generated isolé dans `AI Generated/` plutôt que mélangé avec les notes personnelles
- Tag `source: ai` systématique dans le frontmatter des fichiers générés par Claude
- Sommaires en cascade (racine → sous-sommaires par section) plutôt qu'un seul sommaire monolithique
- `/recap` et `/wrap` mettent à jour leur sommaire respectif automatiquement
- Skip du pipeline d'ingestion vidéo (Kev) — pas pertinent pour le use case actuel
- Skip de Pinecone — vault encore trop petit pour justifier une vector DB
- Sous-sommaires dédiés par type (Sessions/Sommaire.md, Conversations/Sommaire.md) plutôt qu'un seul sommaire AI Generated monolithique
- `/recap` fusionne dans l'existant du jour (append intelligent + dédupe) plutôt que réécriture complète — évite la perte d'info entre mid-session et fin de session
