## Problèmes / Bugs
- **Bug critique non résolu** : le fichier XML sur S3 contient une double déclaration XML (`<?xml ...?><?xml ...?>`) en ligne 1. Le fichier est illisible par un parser XML. Cause probable : le contenu existant sur S3 a déjà une déclaration XML, et le nouvel upload est concaténé ou le bucket renvoie un préfixe. À investiguer côté `uploadXmlToS3` ou état du bucket S3.

## Découvertes
- `getAllInventoryFromBigblue()` retourne `'in stock'` / `'out of stock'` (avec espaces) — format Meta, pas GMC. La conversion vers `in_stock` / `out_of_stock` (underscores) est nécessaire pour le XML GMC et est bien faite dans l'implémentation.
- `invalidateCloudFrontCache` a été ajouté par le linter/formateur automatique dans le code existant (`updateXml`) — l'import existe dans `s3Utils`.
- Le linter a aussi ajouté `.limit(1)` sur la query Mongo dans `generateGmcFeed` (à retirer pour la prod).
- La route est montée sur `/sitemap` dans `server.js`, donc l'endpoint complet est `GET /sitemap/gmc-feed`.

## Décisions prises
- `g:availability` en format underscore pour le XML GMC (`in_stock` / `out_of_stock`), converti depuis la valeur BigBlue (espaces)
- `g:price` = `originalPrice` si discount, `g:sale_price` = prix final calculé
- `g:google_product_category` hardcodée à `Health & Beauty > Skin Care` (générique, affinable plus tard)
- `g:product_type` = catégories DB (`translations.fr.categories`) en complément
- Comportement strict sur la traduction FR manquante : throw et abort (pas de skip silencieux)
- Clé S3 : `feeds/gmc.xml`
