## Découvertes
- **Klaviyo Custom Catalog Feed** supporte XML ou JSON, max 100 MB par fichier (50 MB recommandé), resync automatique **toutes les 6h fixe** (non configurable). Contrainte structurelle "data only one node deep" sauf si feed hébergé comme Google Merchant Feed.
- **Meta Commerce Manager** supporte CSV, TSV, XML (RSS 2.0 / Atom 1.0), XLSX et Google Sheets. Scheduled fetch hourly/daily/weekly.
- **GMC Merchant API DataSources** supporte XML jusqu'à 4 GB via `dataSources.create` avec `fileInput.fetchSettings` (frequency, timeOfDay, timeZone, fetchUri).
- **Format XML compatible avec les 3** : RSS 2.0 avec namespace Google `xmlns:g="http://base.google.com/ns/1.0"` accepté par GMC ET Meta, et toléré par Klaviyo si bien structuré → un seul format peut servir les 3.
- **Klaviyo expose `POST /api/catalog-feeds/{id}/jobs`** pour forcer un sync manuel ad-hoc, utile pour les lancements produits urgents.
- Côté Meta, l'isolation d'un catalogue de test repose **uniquement sur le non-référencement par les ad sets**, pas sur une frontière technique. Discipline d'équipe + nommage `[STAGE]` couvrent le risque résiduel.
- Régénérer le feed XML toutes les heures côté Olis Lab est over-engineered : Klaviyo étant le bottleneck à 6h, un cron toutes les 4h ne dégrade rien et économise Lambda + S3.

## Décisions prises
- **Format XML retenu** : RSS 2.0 + namespace Google `g:*`, un seul fichier sert les 3 plateformes.
- **Pattern de génération du feed** : cron scheduled simple (option 1), pas de hook Payload `afterChange` au démarrage. Évite les régen en boucle pendant les imports bulk. Hybride avec bouton manuel admin (option 3) si latence devient un vrai problème métier.
- **Fréquence de régénération** : toutes les 4h suffit (worst case Klaviyo = 6h, Meta peut absorber l'écart, GMC daily de toute façon).
- **Source de données pour la génération** : Payload Local API ou Mongoose direct depuis le même process. Pas d'appel HTTP à `/api/products` (latence + auth).
- **Pour le POC sandbox** : juste un script `pnpm regenerate-feed` lancé à la main, pas besoin d'EventBridge tant que le feed XML n'est pas validé sur les 3 sandboxes.
