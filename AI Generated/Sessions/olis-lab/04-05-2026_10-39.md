# Recap — olis-lab — 04/05/2026

## Avancées
- Recherche complète sur les structures XML attendues par GMC, Meta Commerce Manager et Klaviyo
- Confirmation que GMC et Meta partagent le même format RSS 2.0 avec namespace `xmlns:g="http://base.google.com/ns/1.0"`
- Identification de la seule divergence GMC/Meta : valeurs `availability` avec underscore (`in_stock`) vs espace (`in stock`)
- Confirmation que Klaviyo requiert un format plat (flat XML/JSON) incompatible avec le feed RSS 2.0 GMC
- Rédaction du plan d'architecture pour les trois feeds (GMC, Meta, Klaviyo)
- Identification de l'architecture finale : 3 URLs S3/CloudFront issues d'une seule source Payload

## Problèmes / Bugs
- Le lien partagé (`developers.google.com/product-review-feeds/sample`) concernait les **avis clients** (product reviews), pas le catalogue produits — hors sujet confirmé
- Klaviyo ne peut pas consommer directement le feed RSS 2.0 GMC (structure nested rejetée), sauf si hébergé sur Google Merchant Feeds — ce qui ne s'applique pas à notre architecture S3

## Découvertes
- Meta accepte le format Google Shopping XML (même namespace), seule différence : valeurs `availability` avec espace
- Klaviyo : le premier item du feed doit contenir **tous** les champs disponibles (utilisé pour le mapping automatique)
- Klaviyo : `price` en décimal pur sans devise (`29.90`, pas `29.90 EUR`)
- Klaviyo : resync toutes les 6h, taille max recommandée 50 MB
- GMC : image minimum 500x500px obligatoire à partir du 31 janvier 2027
- L'exception Klaviyo/Google Merchant Feeds ne s'applique pas à une architecture S3+CloudFront custom

## Décisions prises
- Architecture retenue : **3 fichiers XML distincts** depuis une seule source Payload
  - `feed-gmc.xml` → Google Merchant Center (RSS 2.0, availability avec underscore)
  - `feed-meta.xml` → Meta Commerce Manager (RSS 2.0, availability avec espace)
  - `feed-klaviyo.xml` → Klaviyo (flat XML, prix décimal, sans namespace)
- Les trois feeds hébergés sur S3 + CloudFront
- Pas d'option "un seul fichier" : les contraintes Klaviyo (flat) et la divergence GMC/Meta rendent un feed universel impossible sans risque
