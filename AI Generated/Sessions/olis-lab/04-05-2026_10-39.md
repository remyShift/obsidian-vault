## Problèmes / Bugs
- Le lien partagé (`developers.google.com/product-review-feeds/sample`) concernait les **avis clients** (product reviews), pas le catalogue produits — hors sujet confirmé
- Klaviyo ne peut pas consommer directement le feed RSS 2.0 GMC (structure nested rejetée), sauf si hébergé sur Google Merchant Feeds — ce qui ne s'applique pas à notre architecture S3
- `ActiveArticleCard` utilise des classes CSS globales de `ActivesList/style.scss` : réutilisation directe hors de sa page impossible sans couplage — résolu en créant `ActiveSearchCard` avec SCSS module
- `MappedProduct.translations` attend `TTranslations` mais `BundleSearchResult.translations` est `Record<string, BundleSearchTranslation>` — résolu par cast `as unknown as TTranslations` (compatible à l'usage, seul `ProductCard` lit `translations['en'].title` pour analytics)

## Découvertes
- Meta accepte le format Google Shopping XML (même namespace), seule différence : valeurs `availability` avec espace
- Klaviyo : le premier item du feed doit contenir **tous** les champs disponibles (utilisé pour le mapping automatique)
- Klaviyo : `price` en décimal pur sans devise (`29.90`, pas `29.90 EUR`)
- Klaviyo : resync toutes les 6h, taille max recommandée 50 MB
- GMC : image minimum 500x500px obligatoire à partir du 31 janvier 2027
- L'exception Klaviyo/Google Merchant Feeds ne s'applique pas à une architecture S3+CloudFront custom
- `activesApi.getActivesSearch` n'a pas de paramètre `lang` : les noms d'actives sont language-agnostic, queryKey sans `lang`
- `BundlesApi.getBundlesSearch` retourne `{ bundles: [...] }` (confirmé dans le controller serveur)
- `mapBundleProducts` attend `ShopProduct[]` — pas directement compatible avec `BundleSearchResult[]` sans cast, d'où le mapper inline dans `BundleResults`
- `onAddToBag` pour les bundles dans la search navigue vers `/bundle/${slug}` : le vrai add-to-bag nécessite `checkBundleAvailability` depuis la page de détail

## Décisions prises
- Architecture retenue : **3 fichiers XML distincts** depuis une seule source Payload (feeds GMC, Meta, Klaviyo)
- Les trois feeds hébergés sur S3 + CloudFront
- Pas d'option "un seul fichier" : les contraintes Klaviyo (flat) et la divergence GMC/Meta rendent un feed universel impossible sans risque
- **Pattern map pour le rendu des onglets** : `tabContent[activeTab] ?? null` — plus besoin d'ajouter un bloc conditionnel pour chaque nouvelle collection
- **`ProductsGrid` pour les bundles** : réutilisation via mapper inline plutôt que composant card custom — cohérence visuelle avec les produits
- **`ActiveSearchCard` isolé** : SCSS module propre plutôt que réutilisation de `ActiveArticleCard` avec ses classes globales
