## Découvertes
- Le repo GitHub olis-lab = `olis-lab/web-app`. Les commentaires de review IA de Diego arrivent en **issue comments**, signés "🤖 This review was conducted by Claude".
- Distinction des deux statuts produit : `_status` (Payload draft/published) vs `status` (champ commercial `Live`/`Staged`/`Offline`). Le lock par défaut se base sur `_status`, le blocage de régénération SKU sur `status === 'Live'`.
- Les classes scss du slug natif (`.slug-field-component .label-wrapper`, `.lock-button`) sont dans le bundle `@payloadcms/ui` (chargé avec l'admin) → réutilisables sur un composant custom, contrairement à un `.scss` local importé dans un composant client (refusé par Next hors layout).
- Le `Button` Payload a un `margin-block` par défaut (jusqu'à `var(--base)*1.2`) ; le scss `.slug-field-component .lock-button { margin: 0 }` le neutralise → plus besoin de `margin={false}` une fois les classes natives réutilisées.
- **(Session 17:32) posthog-node + `onlyEvaluateLocally: true`** = renvoie `undefined` si les définitions de flags ne sont pas encore chargées (chargement async non-awaité après création du client, polling). En serverless/cold start, la 1re requête évalue avant le poll → `undefined`. Retirer `onlyEvaluateLocally` fait retomber le SDK sur l'endpoint distant (self-heal), puis local une fois pollé.
- **Éval serveur d'un flag avec un `distinctId` constant** = déterministe pour tous (un rollout par pourcentage deviendrait all-or-nothing), parfait pour un interrupteur de migration on/off, et indépendant de tout cookie. Couplage du flag posthog au cookie de consentement = anti-pattern pour du contenu public non personnalisé.
- **`NODE_ENV`** est le vrai signal local vs déployé : `next dev` → `development`, build Amplify → `production`. Auto-réglé, zéro env var (`NEXT_PUBLIC_ENV` n'était défini nulle part).
- **(cart hook) La CI a une vraie mongo** (`mongo:7`, job `cms-int-tests`) et lance `pnpm test` (int + unit). Le blocage « on ne peut pas QA » concernait l'admin Payload dev (manuel), **pas** la CI → un `.int.spec.ts` est viable et constitue le remplaçant automatisé de la QA manuelle impossible.
- **(cart hook) `cartProductSchema`, `TCartProduct`, `createAsserter` déjà dispo dans `@olis-lab/shared/payload`** ; seul l'export **valeur** `cartProductSchema` a été retiré par le revert (les types restent exportés). La factory `tests/factories/product.ts` a survécu au revert.
- **(cart hook) Le hook ne gère que `operation === 'update'`** ; le create path est commenté (TODO). Cycle réel : create (sans legacyId) → backsync (legacyId) → update → snapshot. D'où « les tests partent d'un produit existant ».
