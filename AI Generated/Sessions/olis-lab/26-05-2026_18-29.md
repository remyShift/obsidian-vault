## Problemes / Bugs
- Le formatter du projet convertit les apostrophes ASCII `'` en guillemets typographiques U+2018/U+2019 dans les nouveaux fichiers, ce qui casse la compilation esbuild — contourne via script Python avec `chr(39)`
- L'import `assertPopulated` depuis `@olis-lab/shared` ne fonctionnait pas car le barrel root exporte `payload` comme namespace (pas de named exports directs)
- Les fonctions standalone `assertPopulated` et `assertMedia` prennent 3 args `(value, entity, path)` mais etaient appelees avec 2 args dans `computeCartSnapshot.ts`

## Decouvertes
- `@olis-lab/shared` exporte le namespace `payload` via `export * as payload from './payload'` — les fonctions ne sont accessibles directement qu'avec `@olis-lab/shared/payload`
- `createAsserter(entity)` fournit des methodes 2-args (`.populated`, `.media`, `.required`) qui bindent l'entity en interne — pattern plus ergonomique quand on a plusieurs assertions pour la meme entite
- Les fonctions `resolveUploadUrl` dans `sync/transformers/brands.ts` et `sync/transformers/products.ts` sont des reimplementations exactes de `assertMedia` — candidats directs au remplacement
- `resolveUploadUrlNullable` a un comportement intentionnellement different (retourne null au lieu de throw) — ne pas remplacer
- `generateBrandTitleSlug` dans `Products.ts` gere les string IDs en fetchant depuis la DB, ce n'est pas le pattern guard — ne pas toucher

## Decisions prises
- Utiliser `assertPopulated` et `assertMedia` en standalone avec les 3 args explicites plutot que `createAsserter` (preference utilisateur)
- `RawProduct` type utilise `AnyRecord = Record<string, unknown>` au lieu des types Payload complets pour permettre des fixtures de test minimales sans casts
- `assert.populated` pour la brand (pas `assert.required`) : verifie que la relation est bien un objet peuple, pas un string ID MongoDB
