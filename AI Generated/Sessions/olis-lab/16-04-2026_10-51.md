## Découvertes
- Les noms d'actives sont traduits dans les fichiers i18n FR (`BEURRE DE KARITÉ`, `ACIDE SALICYLIQUE`) mais `topActives` en DB est toujours en anglais — nécessite de toujours envoyer le titre EN au serveur
- `i18n.getFixedT('en')` est l'API i18next correcte pour obtenir une traduction forcée dans une langue spécifique (vs `i18n.getResource` qui est plus bas niveau)
- Le saut d'écran au changement d'active venait du loading global interceptor, pas du composant lui-même
- Le filtre `imgUrl` côté client dans le hook était redondant car déjà garanti par la query MongoDB

## Décisions prises
- Pas de static method sur le schema Mongoose pour un usage unique — query inlinée directement dans le controller
- Pas de wrapper dans `ProductsApi` — appel `api` direct depuis le hook React Query
- Pas de param `lang` sur l'endpoint — toujours query sur `translations.en` (les actives sont identiques dans toutes les langues en DB)
- Pas de `keepPreviousData` pour l'instant — l'exclusion de l'interceptor suffit à régler le saut visuel
- Permissions lean-ctx en scope user (global) et non projet
