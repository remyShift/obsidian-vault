## Découvertes
- `AbortSignal` est 100% cote client : c'est le navigateur qui coupe la connexion TCP, le backend n'a rien a faire
- React Query fournit automatiquement un `AbortSignal` a chaque `queryFn` — quand la query key change (nouvelle frappe), RQ annule la requete precedente via ce signal passe a axios
- Sans signal : risque de race condition (reponse lente d'une frappe ancienne qui ecrase des resultats plus recents)
- Le plugin `@payload-enchants/translator` fournit le bouton "Translate" dans l'admin UI out of the box — aucun hook custom necessaire pour le cas d'usage principal (manuel)
- Pour l'automatisation au save, il faudrait quand meme un `afterChange` hook custom appelant `translateOperation()` — mais ce n'est pas le workflow retenu
- Le dossier `apps/cms/src/sync/` contient schemas Zod + un transformer Payload → Legacy (sens inverse), mais pas de script de seed Legacy → Payload
- La connexion a la legacy DB est deja configuree (`LEGACY_DB_URI` dans Zod, `src/lib/db/legacy.ts`)

## Décisions prises
- Passer le `signal` a `getArticlesSearch` pour etre consistent avec `getSearchProducts`
- Approche B retenue pour les traductions (plugin communautaire vs hook custom) : moins de code, bouton UI inclus
- `OPENAI_API_KEY` en required dans Zod (pas optional) : fail fast au demarrage si absente plutot qu'un bouton silencieusement casse
- Modele `gpt-4o-mini` (plus recent et moins cher que `gpt-3.5-turbo` deprecie depuis mars 2024)
- Workflow traduction : manuel via bouton (pas d'automatisation au save pour l'instant) — Suze relit toutes les traductions
- Phase 2 optionnelle identifiee : automatisation via `afterChange` hook si besoin futur
