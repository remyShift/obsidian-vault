# Recap — olis-lab — 30/04/2026

## Avancées
- Explication du commentaire de review PR sur l'usage d'`AbortSignal` dans `useSearchPageQuery`
- Ajout du `signal` pour `getArticlesSearch` dans `articleApi.ts` et `useSearchPageQuery.ts` (parity avec `getSearchProducts`)
- Validation que l'implementation articles est correcte : signal destructure depuis `{ signal }` dans `queryFn` et passe a l'API

## Problèmes / Bugs
- Inconsistance initiale : `getSearchProducts` avait le `signal`, `getArticlesSearch` non

## Découvertes
- `AbortSignal` est 100% cote client : c'est le navigateur qui coupe la connexion TCP, le backend n'a rien a faire
- React Query fournit automatiquement un `AbortSignal` a chaque `queryFn` — quand la query key change (nouvelle frappe), RQ annule la requete precedente via ce signal passe a axios
- Sans signal : risque de race condition (reponse lente d'une frappe ancienne qui ecrase des resultats plus recents)

## Décisions prises
- Passer le `signal` a `getArticlesSearch` pour etre consistent avec `getSearchProducts`
