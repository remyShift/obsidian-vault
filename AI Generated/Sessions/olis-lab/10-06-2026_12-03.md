## Découvertes
- Le délai sans feedback a **deux causes**, pas une : (1) côté CRA le `window.location.replace` est une hard navigation, le navigateur garde l'ancienne page figée jusqu'au premier octet de Next ; (2) côté Next, Server Component bloquant sur `getProduct` sans `loading.tsx` ni streaming. Le saut Caddy (`:3005`→`:3002`) est négligeable.
- `getProduct(id, locale)` est appelé **deux fois par rendu** : une fois dans `generateMetadata`, une fois dans `ProductDetailPage` → deux round-trips CMS séquentiels.
- L'atom `isNavigatingAtom` n'a pas besoin d'être remis à `false` : le `window.location.replace` détruit toute l'app CRA (DOM + état JS + atomes), l'overlay disparaît avec la page. Réinitialisé à `false` à chaque nouveau load.

- `git fetch --prune` a nettoyé une grosse pile de remote-tracking refs périmées (branches déjà mergées/supprimées sur GitHub) — ménage local sans impact.
