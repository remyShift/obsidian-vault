## Découvertes
- Le délai sans feedback a **deux causes**, pas une : (1) côté CRA le `window.location.replace` est une hard navigation, le navigateur garde l'ancienne page figée jusqu'au premier octet de Next ; (2) côté Next, Server Component bloquant sur `getProduct` sans `loading.tsx` ni streaming. Le saut Caddy (`:3005`→`:3002`) est négligeable.
- `getProduct(id, locale)` est appelé **deux fois par rendu** : une fois dans `generateMetadata`, une fois dans `ProductDetailPage` → deux round-trips CMS séquentiels.
- L'atom `isNavigatingAtom` n'a pas besoin d'être remis à `false` : le `window.location.replace` détruit toute l'app CRA (DOM + état JS + atomes), l'overlay disparaît avec la page. Réinitialisé à `false` à chaque nouveau load.
- Rename de branche d'une PR ouverte : passer par l'UI/API GitHub (la PR suit le nouveau nom), JAMAIS par delete+recreate du remote (ça ferme la PR).
- `git fetch --prune` a nettoyé une grosse pile de remote-tracking refs périmées (branches déjà mergées/supprimées sur GitHub) — ménage local sans impact.
- Le vrai chemin de la PDP Next est `apps/web/app/[locale]/products/[id]/` (et non `app/product/[id]/` qui apparaît encore dans des overviews périmés).
- Bug "stuck locale" (topbanner/navbar/footer noir en FR sous EN au switch sans refresh) : **rien n'est hardcodé**, tout passe par next-intl avec les clés EN présentes. Cause racine = `NextIntlClientProvider` monté dans le **root layout** (`app/layout.tsx`, locale via cookie `getLocale`/`getMessages`), pas de `[locale]/layout.tsx` → le provider n'est pas re-rendu lors d'un `router.replace` de locale (navigation client), les composants client gardent les anciens messages jusqu'à un refresh complet. Fix minimal possible = `router.refresh()` après `router.replace` dans `Footer/LanguageSelector.tsx` (laissé à Kyle).
- Navbar : `NavItem` rend un `<button>` (curseur flèche par défaut) ; l'item `brands`/"MARQUES" (hasSubmenu:false, hors `NAV_ITEM_ORDER`) est le seul item navbar à naviguer directement → d'où le ressenti "pas de pointer".
- Sections "Your Lab" volontairement désactivées par design (`comingSoon` + `sectionDisabled`) sauf que `personalisedPicks` pointe vers de vraies routes legacy déjà existantes → réactivable sans risque.
- `deliveryTime` (`timedelivery`) vient du backend toujours en anglais ("2-3 days") : le template i18n est traduit mais l'unité, injectée brute dans `{days}`, ne l'était jamais. Le legacy parsait/traduisait l'unité (`checkout.days`).
