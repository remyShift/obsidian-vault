## Decouvertes
- `assert.required` pour les group fields, `assert.populated` pour les relationship fields — distinction importante
- Tant que `howToUse` restait dans `rawRest`, la branche Collectibles ramenait le type brut dans l'union de retour -> `TProductHowToUse` heritait de `step: (string | ProductHowToUseStep)[]`
- `virtual: true` Payload = champ calcule via `afterRead`, non stocke en DB
- `@payloadcms/sdk` supporte `findGlobal()` — pattern identique a `apps/web/lib/api/cms.ts`
- `REACT_APP_PAYLOAD_CMS_URL` absent de `.env.prod`
- `tsc --noEmit` revele des erreurs que vitest/esbuild masquent silencieusement

## Decisions prises
- `skinTypes` et `textureImage` : noms originaux conserves dans l'output du guard (pas de renommage)
- `cartProduct` : field-by-field assertion ; `TCartProduct` dans shared, pas dans web

- Sync curationSettings : Payload -> legacy, declenche manuellement
