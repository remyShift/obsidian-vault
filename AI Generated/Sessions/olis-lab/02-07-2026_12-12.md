## Découvertes
- Accès Mongo brut confirmé par le JSDoc de l'adapter : `payload.db.collections['product-actives'].collection.find({}).toArray()`.
- Resolvers du plugin translator stockés au runtime dans `payload.config.custom.translator.resolvers` (objets complets avec `.resolve`) ; `openAIResolver().key === 'openai'`.
- `resolver.resolve({ localeFrom, localeTo, req, texts })` appelable direct depuis une migration avec `{ payload } as PayloadRequest` ; chunke à 100 en interne, renvoie `translatedTexts` index-alignés.
- Fallback : `fallback: true` + `defaultLocale: 'en'` → un read `locale: 'fr'` vide sert la valeur `en`, y compris sur relations peuplées (`depth > 0`, même locale que la requête). Déclencheur = `null`/absent, **pas** la chaîne vide. Désactivable par requête via `fallbackLocale: 'none'` ; `locale: 'all'` renvoie `{ en, fr }` brut.
- Un champ `text` localisé reste typé `string` dans les types générés (type résolu, pas per-locale).
