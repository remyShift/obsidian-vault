## Problemes / Bugs

## Decouvertes
- Le pattern `null` vs `[]` pour `filteredProducts` etait deja connu (mentionne dans le hot cache comme "Bug PR #1700") mais n'avait pas ete applique systematiquement sur toutes les pages
- Semantique claire : `null` = CategoriesFilter n'a pas encore callbacke ; `[]` = filtre actif sans resultat
- Plusieurs pages utilisaient deja des hooks React Query mais n'exposaient pas `isLoading` — 1 ligne suffit a le corriger
- `filteredProducts ?? products` est plus lisible et semantiquement plus juste que le ternaire `length > 0`

## Decisions prises
- `filteredProducts` type `ShopProduct[] | null` (init `null`) sur toutes les pages concernees
- Spinner inline dans la zone produits pour les pages avec banner independant ; spinner full-page pour EditPage (banner depend du meme fetch)
- Pas d'abstraction commune pour le pattern filteredProducts — chaque page reste autonome (YAGNI)
- Hooks dans `hooks/query/` meme pour des API differentes (CollectionApi, BundlesApi)
