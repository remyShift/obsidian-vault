## Découvertes
- Le flag servait de double garde : feature-flag-off **et** état de chargement. `flagsReady` ne servait qu'à éviter le flash du fallback ; sans fallback, `useFeatureFlags` devient inutile dans Home/Shop.
- `ErrorPage` (`pages/ErrorPage/ErrorPage`) est un composant à export default, déjà routé dans `Routes.tsx` ; rendu inline sur `isError` = pas de changement d'URL ni flicker.
- Les locales es/it sont incomplètes (6 clés orphelines vs 12 pour en/fr) — script Node tolérant aux clés manquantes.
- `HomeBanner`/`Hero`/`BrandOfMonth` prennent exactement `THeroBanner`/`TBrandOfMonth` comme props.
- Le cast `as React.CSSProperties` est nécessaire pour les CSS custom properties (`--bg-img`) : `@types/react` 18.3 ne déclare aucune signature d'index `--*`. Un seul cast suffit (le `['--bg-img' as string]` interne était redondant), sauf que sans cast du tout → TS2559 via framer-motion (weak type `MotionStyle`).
- Pattern asserter partagé du repo : `createAsserter(entity)` (`packages/shared/src/payload/guard.ts`) → `.populated/.required/.media` ; `assertMedia` garantit `url: string` (`ResolvedMedia<T>`). Convention : asserter par entité (`assertProduct`, `assertTradingPlan`), type dérivé `ReturnType<typeof assert...>`, appelé dans la couche fetch.
- **web_client consomme le `dist` de `packages/shared`** (via exports `./payload`) → rebuild obligatoire après modif (même piège que le cms).
- Tests vitest shared : `include: src/**/*.test.ts`, dossier `src/payload/tests/`.
