## Problèmes / Bugs
- Le formatter (Prettier) convertissait les commentaires HTML `<!-- ... -->` en `# <!-- ... -->` a chaque save, cassant la structure du fichier — resolu via `.prettierignore`
- Le formatter revertait aussi les corrections des props SkillCompare (remettait des lignes vides entre les props et `> <template`), necessitant de refixer 3 fois — resolu en mettant toutes les props sur une ligne
- Les `<style>` Slidev ne sont pas scopes par slide mais **globaux** : les slides 26 et 27 redefinissaient toutes les deux `h2 { ... }` en global, conflit silencieux — corrige avec classe `.slide-h2-centered`

## Découvertes
- `## layout: xxx` dans le frontmatter Slidev est parse comme un **commentaire YAML** (le `#` est le separateur de commentaire YAML) — le layout n'est donc jamais applique si ecrit avec `##`
- Les `<style>` dans les slides Slidev sont **globaux**, pas scopes — redefinir `h2 {}` dans deux slides differentes cree un conflit silencieux
- Le pattern `> <template #slot>` est une tentative de mixer blockquote markdown et slot Vue — ca casse le compilateur Vue systematiquement, la seule solution est d'ecrire les props sur une seule ligne ou d'utiliser du HTML pur sans ligne vide avant le template

## Décisions prises
- `.highlight-orange`, `.statement-text`, `.slide-h2-centered` → classes globales dans `styles/index.css`, pas inline dans les slides
- Props SkillCompare sur une seule ligne pour immuniser contre le formatter
- `.prettierignore` sur `slides.md` plutot que config Prettier complexe
- L'intro V1 (note) est retenue : hook direct + liste du parcours revele en v-click — l'anecdote du mardi matin reste dans l'Acte 1 slide 7 (setup vente)
