# Recap — Portfolio-GameBoy-Next — 24/04/2026

## Avancées

- **Effet de profondeur sur l'écran GameBoy (Alt 4 combo)** : CSS inset shadows renforcees sur `Screen.tsx` (creux statique permanent) + parallax du contenu interne dans un wrapper autour de `{children}`. Le cadre GameBoy reste totalement immobile, seul le contenu glisse avec le curseur. Biseau wine ajoute sur `WineBorder.tsx` (inset shadow double : sombre en haut, highlight en bas).
- **Hook `useMouseParallax` cree** : normalise la position souris en `{x, y}` en -1..1 relatif au viewport, throttle rAF, respecte `prefers-reduced-motion`. 4 tests de pure function + 4 tests de hook wiring.
- **Boost de sensibilite** : ajout d'une courbe de saturation (`SENSITIVITY = 1.6`, `boost(v) = clamp(v * SENSITIVITY, -1, 1)`) pour que la parallaxe soit visible au mouvement normal (saturation a ~60% du viewport) et pas seulement en collant les bords. Amplitude `CONTENT_PARALLAX_PX = 20`.
- **Support tilt mobile (tentative puis revert)** : hook etendu avec `computeOrientationParallax` + branchement `deviceorientation`, detection `matchMedia("(hover: hover)")`, permission iOS via `DeviceOrientationEvent.requestPermission` sur premier `pointerdown`, calibration 1re lecture = origine. 22 tests. Puis revert complet apres test iPhone infructueux.
- **Simplification mobile** : hook retourne `{0, 0}` immediatement sur `(hover: none)` sans attacher aucun listener. Zero transform sur mobile.
- **BackgroundSign centrage** : `w-full` remplace par `flex-1` dans `BackgroundSign.tsx`. Corrige le decalage de 28px vers la droite sur mobile (sign etait ~47% de large au lieu de 1/3).
- **Fix audio boot (round 1)** : appel `playWelcomeChime` deplace dans le `.then()` de `ctx.resume()` pour que `ctx.currentTime` soit valide quand les notes sont schedulees.
- **Fix audio boot (round 2)** : double declencheur `statechange` + `.then()` avec flag `played` pour contourner le bug WebKit ou `resume()` ne resout parfois jamais sa Promise. Offset scheduling `retroAudio.ts` passe de 20ms a 80ms pour laisser le moteur audio mobile demarrer.

## Problèmes / Bugs

- **Premier round parallaxe : effet invisible** : normalisation lineaire sur viewport entier → souris au-dessus de la GameBoy = tres pres du centre = shift effectif 1-5px. Les 30px max n'etaient atteints qu'aux bords extremes.
- **Mauvais modele mental** : l'utilisateur attendait que le contenu bouge DANS l'ecran, pas que le cadre entier bouge. Premier plan (parallaxe sur `WineBorder` vs `Screen`) faisait bouger le cadre avec le contenu → aucune sensation de profondeur.
- **Firefox : parallaxe morte** : `matchMedia("(prefers-reduced-motion: reduce)").matches` retournait `true` sur Firefox mais `false` sur Chrome. Cause : `privacy.resistFingerprinting` active dans les settings Firefox de Remy, qui force `prefers-reduced-motion: reduce` sur tous les sites.
- **iOS permission gyro non declenchee** : sur iPhone, la popup `requestPermission` ne s'affiche pas sur le dev server HTTP local. iOS Safari exige HTTPS strict pour `DeviceOrientationEvent.requestPermission()` — silencieux sans erreur console.
- **BackgroundSign decale** : `w-full` sur un enfant direct d'un flex-row sans `w-1/3` = ~47% de largeur a cause de flex-shrink, au lieu du 1/3 attendu.
- **Son de boot : 1 fois sur 2 sur mobile** : meme apres fix round 1, le chime manque aleatoirement. Cause identifiee : `AudioContext.resume()` retourne parfois une Promise qui ne resout jamais sur WebKit. Fix round 2 applique mais bug persiste selon le dernier test utilisateur (accepte pour le moment).

## Découvertes

- Saturation non-lineaire indispensable pour parallaxe viewport-relative : sans boost, le ressenti est aplati car l'utilisateur reste en pratique dans les 20% centraux du viewport.
- `privacy.resistFingerprinting` de Firefox ment sur `prefers-reduced-motion` pour ameliorer l'anonymat. Tres commun chez les utilisateurs privacy-conscious, et casse silencieusement toute animation cosmetique respectueuse de l'a11y.
- iOS Safari exige HTTPS pour `DeviceOrientationEvent.requestPermission()` — pas testable en dev local HTTP, meme via IP LAN. Necessite ngrok, Caddy, ou preview Vercel.
- WebKit bug connu : `AudioContext.resume()` peut retourner une Promise qui ne resout jamais. Solution robuste : combiner `statechange` listener + `.then()` avec un flag de deduplication.
- Sur mobile, le moteur audio a besoin de ~80ms apres `resume()` avant que `ctx.currentTime` avance de facon fiable. Scheduler a 20ms de lookahead = notes schedulees dans le passe = silence.
- `w-full` vs `flex-1` dans un parent flex : `w-full` declenche flex-shrink et peut donner moins de largeur que prevu selon les voisins. `flex-1` = "je prends exactement l'espace restant" = plus previsible.
- Structure confirmee : `WineBorder` n'est PAS une vraie border, c'est juste un `div bg-wine` 5% plus grand que `Screen` qui depasse autour. Pour une profondeur visible, cibler une couche interne au Screen, pas le Screen lui-meme.

## Decisions prises

- **Revert integral du tilt mobile** : la popup iOS ne se declenche pas en dev HTTP, et Remy veut avancer sans s'enliser. Code d'orientation supprime entierement plutot que garde en dead code. Reimplementable plus tard via PR dediee en HTTPS preview Vercel.
- **Pas de contournement `prefers-reduced-motion`** : respecter la preference explicite est la bonne pratique. La solution pour Remy = corriger `privacy.resistFingerprinting` dans Firefox, pas bidouiller le code.
- **Alt 4 retenu** (CSS depth statique + parallaxe contenu) : depth visible des le premier paint (mobile + reduced-motion inclus), effet vivant au mouvement sur desktop, zero risque sur le cadre GameBoy, le boot, les ombres.
- **Mobile = zero parallaxe** : shortcut `hover: none → return early` dans le hook. Plus simple que deux canaux d'input a maintenir, aligne avec le fait que la parallaxe desktop ne s'etend pas naturellement au tactile.
- **Son chime boot : double declencheur robuste** : `statechange` + `.then()` + flag `played`. Verbose mais contourne proprement le bug WebKit. Offset `ATTACK_LOOKAHEAD_MS` passe a 80ms.
