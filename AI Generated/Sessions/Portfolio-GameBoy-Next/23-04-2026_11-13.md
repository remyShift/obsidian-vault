# Recap — Portfolio-GameBoy-Next — 23/04/2026

## Avancées

- **i18n FR/EN** : next-intl v4 integre avec `localePrefix: 'as-needed'` (FR sans prefix, EN avec `/en/`). Toutes les pages traduites (nav, welcome, about, contact, projects, settings, footer, breadcrumb, metadata, error).
- **Boot animation** : animation GameBoy CRT avec phases (crt → content → wait-for-input), lettres animees une par une, scanlines, "PRESS START" clignotant. Ne se ferme plus automatiquement — attend un appui clavier ou tactile.
- **Sons retro** : module `retroAudio.ts` centralise. Melodie de boot (5 notes triangle), son de clic navigation (triangle court 520→380 Hz, ~35 ms). `NavClickSound` client component monte dans le root layout. Debloquage AudioContext lie au geste utilisateur (politique autoplay respectee).
- **Curseur pixel art** : `PixelCursor.tsx` — fleche pixel creme sur fond sombre. Sur hover element cliquable : curseur custom masque, `cursor: pointer` natif restaure (via CSS `button, a { cursor: pointer !important }`). Pas de double curseur.
- **Page Settings** : nouvelle route `/settings`, structure 3 colonnes (label | BackgroundSign | controles), `LanguageSetting` client component avec selecteur `► Francais / English` style menu GameBoy, extensible (ajouter un `<li>` dans chaque colonne).
- **Flash de boot corrige** : `useState(true)` par defaut dans `Screen.tsx` — SSR rend l'overlay immediatement, `useLayoutEffect` le masque avant le premier paint si `boot-seen` est dans sessionStorage. Fonctionne sur nouvel onglet et redirect.
- **Padding/spacing Screen** : suppression du reset global `* { margin:0; padding:0 }` hors `@layer base` qui ecrasait silencieusement toutes les utilities Tailwind de spacing. Mis dans `@layer base` pour respecter la cascade.
- **BackgroundSign** : `w-full` au lieu de `w-1/3` dans le div interne — corrige l'affichage quand le composant est place dans une colonne deja `w-1/3`.
- **Mobile tactile boot** : handlers `onPointerDown` + `onTouchStart` directement sur le `motion.div` overlay avec `dismissingRef` pour eviter le double-declenchement.
- **Bruit de fond (noise)** : `pointer-events: none` ajoute sur le `span` lui-meme (pas seulement le `::before`) pour ne pas intercepter les touches sur mobile.

## Problemes / Bugs

- **ESLint `react-hooks/set-state-in-effect`** : `setShowBoot(true)` dans `useEffect` declenchait la regle. Resolu en passant a `useLayoutEffect` (la regle ne s'applique pas a ce hook).
- **Hydration mismatch** : `useState(false)` causait un flash visible entre le rendu SSR (sans overlay) et l'hydratation. Resolu avec `useState(true)` + `useLayoutEffect` qui masque avant le premier paint.
- **Double curseur** : `cursor: none` sur `html` ne cascadait pas sur les balises natives (`a`, `button`). Fix : `*, *::before, *::after { cursor: none !important }` puis exceptions explicites `button, a, ... { cursor: pointer !important }`.
- **Clic sur hover interne** : les `<span>` a l'interieur des `<button>` matchaient `* { cursor: none }` sans etre couverts par `button { cursor: pointer }`. Fix CSS etendu avec `button *, a *, ...`.
- **AudioContext autoplay bloque** : premier chargement sans geste = son muet. Gere proprement : tentative au montage, replay au premier `pointerdown` sur l'overlay si le contexte est encore `suspended`. Pas de hack.
- **Tactile mobile boot** : `ref` sur `motion.div` parfois `null`, les `addEventListener` ne se branchaient sur rien. Resolu en utilisant les handlers React (`onPointerDown`, `onTouchStart`) directement sur le noeud motion.
- **Bouton plein ecran (tentative echouee)** : ajout d'un `<button>` en recouvrement + wrapper `z-100` decalait et cassait le layout. Revenu a la structure d'origine avec handlers directement sur le `motion.div`.

## Decouvertes

- `useLayoutEffect` s'execute de facon synchrone apres la mise a jour du DOM mais avant le premier paint du navigateur — indispensable pour eviter les flashs d'UI bases sur `sessionStorage`.
- `localePrefix: 'as-needed'` avec next-intl : `usePathname()` retourne toujours le chemin sans prefixe locale (ex. `/en/about` → `/about`), donc `VALID_PATHS` reste inchange.
- Le reset global `* { margin:0; padding:0 }` hors `@layer` a une priorite CSS superieure aux `@layer utilities` de Tailwind — il ecrase silencieusement tous les spacing utilities sans warning.
- `BackgroundSign` avec `w-1/3` a l'interieur d'un parent `w-1/3` = element a 1/9 de la largeur totale (perte de l'effet decoratif).
- Sur iOS/WebKit, les `addEventListener` sur un `motion.div` via `ref` sont parfois ignores. Les handlers React (`onPointerDown`, `onTouchStart`) sur le noeud motion sont plus fiables.
- La politique autoplay navigateur est incontournable de facon propre. La solution standard : lier le premier son a un geste utilisateur explicite et informer l'utilisateur si le son est bloque.

## Decisions prises

- **Curseur hover** : retour au `cursor: pointer` natif sur les elements cliquables (pas de SVG custom pour pointer state — trop difficile a rendre proprement en pixel art a petite taille).
- **Boot "press to continue"** : suppression de la fermeture automatique — l'utilisateur doit interagir, ce qui debloque aussi l'AudioContext naturellement.
- **Settings extensibles** : structure 2 colonnes synchronisees (`<ul>` labels + `<ul>` controles) avec `gap` identique — ajouter un setting = ajouter un `<li>` dans chaque colonne.
- **Sons** : module centralise `retroAudio.ts`, onde triangulaire (plus douce que carre), volumes tres bas (0.0055 clic, 0.04 melodie). `NavClickSound` ignore l'overlay du boot (`[data-boot-overlay]`).
- **i18n routing** : `proxy.ts` (renommage Next.js 16 de `middleware.ts`), API route deplacee de `/contact/api/send` vers `/api/contact/send` pour eviter le conflit avec `[locale]`.
- **Pas de 3D dans ce PR** : le GameBoy 3D (react-three-fiber) est reserve a un PR separee depuis main.
