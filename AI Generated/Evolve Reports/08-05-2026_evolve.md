**Dossier cible** : `Notes Permanentes/Code/Dev Knowledge/Webdev knowledge/`
**Source** : Conversation 29/04 "Débat SEO dans Layout vs Page component"
**Justification** : Pattern d'architecture Next.js durable, validé par Kyle Conrad. "You're confusing the concept with the component — SEO is infra, not business logic."
**Contenu proposé** :
```markdown
---
tags: [next-js, architecture, seo, components]
---

# Next.js : SEO dans Layout, données dans Page

## Le pattern

- **Layout component** : SEO (generateMetadata, title, meta) — infrastructure transversale
- **Page component** : data fetching (React Query, fetch) — logique métier

## Pourquoi

SEO n'est pas une donnée de la page, c'est une infrastructure. Confondre les deux mélange présentation
et configuration système.

Kyle Conrad (30/04/2026) : "You're confusing the concept with the component. SEO is infra, not
business logic."

## Exemple (SearchPage olis-lab)

// SearchLayout.tsx — infra SEO ici
export function generateMetadata(): Metadata {
  return { title: '...', description: '...' }
}

// SearchPage.tsx — data fetching ici
function SearchPage() {
  const { data } = useSearchPageQuery(query)
  return <SearchResults ... />
}

## Pattern associé

Pagination via query params : Page lit searchParams, passe au hook React Query.
```
