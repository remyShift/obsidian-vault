### [V1] nouvelle-note — skipToken (TanStack Query v5)
**Dossier cible** : `Notes Permanentes/Code/Dev Knowledge/Webdev knowledge/` (ou `React Query/` si existant)
**Source** : Session olis-lab 20/05/2026
**Justification** : Pattern technique réutilisable sur tout projet TanStack Query v5. Remplace le pattern `enabled: !!param + param!` qui force une assertion non-null. Connaissance durable, hors contexte olis-lab.
**Contenu proposé** :
```markdown
---
tags: [tanstack-query, react-query, typescript, pattern]
---

# skipToken — TanStack Query v5

`skipToken` permet de skip une query lorsqu'un paramètre est nullable, sans avoir à combiner
`enabled: !!param` et une assertion non-null `param!` dans le `queryFn`.

```ts
// Avant (v4 pattern, verbeux et unsafe)
useQuery({
  queryKey: ['resource', id],
  queryFn: () => fetchResource(id!),
  enabled: !!id,
})

// Après (v5 avec skipToken)
import { skipToken } from '@tanstack/react-query'

useQuery({
  queryKey: ['resource', id],
  queryFn: id ? () => fetchResource(id) : skipToken,
})
```

Quand `skipToken` est passé comme `queryFn`, la query est mise en pause ET TypeScript narrow
automatiquement le type dans le corps du `queryFn` — plus besoin de `!`.

**Inutile pour** les hooks sans paramètre nullable (ex: useAllBundlesQuery) — utiliser `skipToken`
uniquement quand le paramètre peut être `null` ou `undefined`.
```
